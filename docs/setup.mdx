# Setup

This page describes the current local development setup for a Kozz environment with the gateway, the Baileys WhatsApp boundary and a module project such as Gramonta Bot.

## Clone the Repositories

```bash
mkdir my-kozz-workspace
cd my-kozz-workspace

git clone git@github.com:Kozz-Bot/Kozz-Gateway.git kozz-gw
git clone git@github.com:Kozz-Bot/kozz-baileys.git kozz-baileys
git clone git@github.com:Kozz-Bot/Gramonta-Bot.git kozz-gramonta-bot
```

Install dependencies in each repository:

```bash
cd kozz-gw && bun install
cd ../kozz-baileys && bun install
cd ../kozz-gramonta-bot && bun install
```

## Generate Gateway Keys

The gateway authenticates boundaries and modules using an RSA key pair.

```bash
cd kozz-gw
./scripts/generate_key_pair.sh
```

On Windows, use:

```powershell
cd kozz-gw
./scripts/generate_key_pair.ps1
```

The command creates:

```text
kozz-gw/
└── keys/
    ├── publickey.pem
    └── privatekey.pem
```

Keep `publickey.pem` in the gateway. Copy `privatekey.pem` into every service that must authenticate with the gateway:

```text
kozz-baileys/keys/privatekey.pem
kozz-gramonta-bot/keys/privatekey.pem
```

## Configure Environment Variables

The most common variables are:

```bash
GATEWAY_URL=ws://localhost:4521
BOUNDARY_NAME=kozz-baileys
SOCKET_PATH=
```

`BOUNDARY_NAME` matters because modules use it when they ask resources or send messages through a specific boundary. Some older deployments still use names such as `Gramonta-Wa`; keep the configured name consistent across the boundary and modules.

## Start the Services

Start the gateway:

```bash
cd kozz-gw
bun run start
```

Start the WhatsApp boundary:

```bash
cd kozz-baileys
bun run start
```

The boundary will ask you to authenticate with WhatsApp. Scan the QR code from the terminal output.

Start your module project:

```bash
cd kozz-gramonta-bot
bun run start
```

## Development Commands

The projects use slightly different build scripts:

```bash
# Gateway
cd kozz-gw
bun run start

# Baileys boundary
cd kozz-baileys
bun run start

# Gramonta Bot
cd kozz-gramonta-bot
bun run build
bun run start
```

For type-checking during development:

```bash
cd kozz-baileys
./node_modules/.bin/tsc --project tsconfig.json

cd ../kozz-gramonta-bot
./node_modules/.bin/tsc --project tsconfig.json --noEmit
```
