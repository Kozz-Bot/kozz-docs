# Setup

This page describes a local development setup for the Kozz framework: one gateway, one boundary and one custom module project.

## Clone Runtime Services

Create a workspace and clone the gateway plus the Baileys WhatsApp boundary:

```bash
mkdir my-kozz-workspace
cd my-kozz-workspace

git clone git@github.com:Kozz-Bot/Kozz-Gateway.git kozz-gw
git clone git@github.com:Kozz-Bot/kozz-baileys.git kozz-baileys
```

Install dependencies:

```bash
cd kozz-gw && bun install
cd ../kozz-baileys && bun install
```

## Generate Gateway Keys

The gateway authenticates boundaries and modules using an RSA key pair.

```bash
cd kozz-gw
./scripts/generate_key_pair.sh
```

On Windows, use:

```powershell
cd kozz-gw
./scripts/generate_key_pair.ps1
```

The command creates:

```text
kozz-gw/
└── keys/
    ├── publickey.pem
    └── privatekey.pem
```

Keep `publickey.pem` in the gateway. Copy `privatekey.pem` into every boundary and module project that must authenticate with the gateway:

```text
kozz-baileys/keys/privatekey.pem
my-modules/keys/privatekey.pem
```

## Configure the Boundary

The most common boundary variables are:

```bash
GATEWAY_URL=ws://localhost:4521
BOUNDARY_NAME=kozz-baileys
SOCKET_PATH=
```

`BOUNDARY_NAME` matters because modules use it when they ask resources or send messages through a specific boundary. Keep the configured boundary name consistent across services.

## Start the Gateway and Boundary

Start the gateway:

```bash
cd kozz-gw
bun run start
```

Start the WhatsApp boundary:

```bash
cd kozz-baileys
bun run start
```

The boundary will ask you to authenticate with WhatsApp. Scan the QR code from the terminal output.

## Create a Module Project

Create a separate project for your custom modules:

```bash
cd my-kozz-workspace
mkdir my-modules
cd my-modules
bun init
bun add kozz-module-maker
bun add -d kozz-types typescript
mkdir keys src
```

Copy the private key generated by the gateway:

```bash
cp ../kozz-gw/keys/privatekey.pem ./keys/privatekey.pem
```

Create a minimal module:

```ts title="src/index.ts"
import { createMethod, createModule } from 'kozz-module-maker';

const ping = createMethod('default', requester => {
  requester.reply('pong!');
});

createModule({
  name: 'ping',
  address: process.env.GATEWAY_URL ?? 'ws://localhost:4521',
  customSocketPath: process.env.SOCKET_PATH,
  commands: {
    boundariesToHandle: ['*'],
    methods: {
      ...ping,
    },
  },
});
```

Run the module:

```bash
bun run src/index.ts
```

Then send this message in a connected chat:

```text
!ping
```

The module should reply with `pong!`.

## Type Checking

For the runtime services:

```bash
cd kozz-baileys
./node_modules/.bin/tsc --project tsconfig.json
```

For a custom module project, use your own TypeScript configuration. A minimal check is:

```bash
./node_modules/.bin/tsc --noEmit
```
