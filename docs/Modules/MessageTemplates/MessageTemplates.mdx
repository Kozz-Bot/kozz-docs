# Message Components

Message templates are now written as JSX/TSX components. This replaces the older `.kozz.md` template files.

The old `templatePath`, `loadTemplates` and `requester.reply.withTemplate(...)` flow is considered legacy. Prefer regular TypeScript components rendered directly in replies.

## Why TSX Components?

TSX message components let you:

- keep message rendering close to module code;
- use TypeScript props for template data;
- share helpers across messages;
- render lists and conditionals with normal code;
- avoid a custom markdown-like template syntax.

## Setup

`kozz-module-maker` exports a small JSX runtime and message formatting components.

For projects compiled with Babel, configure React automatic runtime with `kozz-module-maker/dist` as the import source:

```json title=".babelrc"
{
  "presets": [
    "@babel/preset-typescript",
    [
      "@babel/preset-react",
      {
        "runtime": "automatic",
        "importSource": "kozz-module-maker/dist"
      }
    ]
  ]
}
```

In `tsconfig.json`, keep JSX syntax available to your build pipeline:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "jsx": "preserve"
  }
}
```

If you use another build tool, configure it to use the automatic JSX runtime from `kozz-module-maker/dist`.

## Basic Message

Create a component:

```tsx title="src/messages.tsx"
import { Bold, Line } from 'kozz-module-maker';

export const Greeting = ({ name }: { name: string }) => {
  return (
    <>
      <Line>
        Hello, <Bold>{name}</Bold>!
      </Line>
      <Line>Welcome to Kozz Bot.</Line>
    </>
  );
};
```

Use it inside a command:

```tsx title="src/index.tsx"
import { createMethod, createModule } from 'kozz-module-maker';
import { Greeting } from './messages';

const hello = createMethod('hello', requester => {
  requester.reply(
    <Greeting name={requester.message.contact.publicName || 'there'} />
  );
});

createModule({
  name: 'demo',
  address: 'ws://localhost:4521',
  commands: {
    boundariesToHandle: ['*'],
    methods: {
      ...hello,
    },
  },
});
```

Calling `!demo hello` replies with the rendered message.

## Formatting Components

`kozz-module-maker` currently exports these message components:

- `Line`: renders one line and appends a line break.
- `Bold`: renders bold text.
- `Italic`: renders italic text.
- `Stroke`: renders strikethrough text.
- `Code`: renders a multi-line monospace block.
- `Monospace`: renders inline monospace text.
- `Paragraph`: renders a bold paragraph-like heading.
- `ListItem`: renders an indented list item.
- `List`: renders one or more list items from strings.
- `ForEach`: renders a component for each item in an array.
- `Render`: conditionally renders children.

Example:

```tsx
import { Bold, Code, ForEach, Line, ListItem, Render } from 'kozz-module-maker';

type RankingEntry = {
  name: string;
  points: number;
};

export const Ranking = ({ entries }: { entries: RankingEntry[] }) => {
  return (
    <>
      <Line>
        <Bold>Ranking</Bold>
      </Line>
      <Render when={entries.length === 0}>
        <Line>No entries yet.</Line>
      </Render>
      <Render when={entries.length > 0}>
        <Code>
          <ForEach
            data={entries}
            render={(entry, index) => (
              <Line>
                {index + 1}. {entry.name.padEnd(20, ' ')} {entry.points}
              </Line>
            )}
          />
        </Code>
      </Render>
    </>
  );
};
```

For a simpler list:

```tsx
import { ForEach, ListItem } from 'kozz-module-maker';

export const Commands = ({ commands }: { commands: string[] }) => (
  <>
    <ForEach
      data={commands}
      render={command => <ListItem>{command}</ListItem>}
    />
  </>
);
```

## Replying With Components

Every text reply accepts rendered JSX:

```tsx
requester.reply(<Greeting name="Ada" />);
```

You can also send generated component text through other message APIs that accept a string body, because the JSX runtime resolves to text.

## Help Resources

If your module exposes a `help` resource, the getter can return a component:

```tsx
import { Bold, Line } from 'kozz-module-maker';

const Help = () => (
  <>
    <Line>
      <Bold>demo</Bold>
    </Line>
    <Line>!demo hello - sends a greeting</Line>
  </>
);

createModule({
  name: 'demo',
  address: 'ws://localhost:4521',
}).resources.upsertResource('help', () => <Help />);
```

## Legacy `.kozz.md` Templates

Older modules may still use:

```ts
templatePath: 'src/messages.kozz.md'
requester.reply.withTemplate('Help')
```

This API is kept for compatibility with existing modules, but new modules should not use it. Prefer TSX message components and direct replies.

When migrating, move each template into a typed component:

```md
> @Greeting

# Hello {{name}}

> ---
```

becomes:

```tsx
import { Bold, Line } from 'kozz-module-maker';

export const Greeting = ({ name }: { name: string }) => (
  <Line>
    <Bold>Hello {name}</Bold>
  </Line>
);
```
