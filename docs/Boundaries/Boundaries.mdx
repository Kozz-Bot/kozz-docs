# Boundaries

A boundary is a service that connects Kozz Bot to a chat platform.

It owns the platform SDK, translates platform-specific payloads into Kozz payloads, and handles gateway requests such as sending messages, replying, reacting and deleting.

The gateway does not know how WhatsApp, Discord, Telegram or any other platform works. It only knows the boundary protocol.

## Responsibilities

A boundary is responsible for:

- connecting to one chat platform;
- introducing itself to the gateway;
- emitting normalized messages;
- handling outbound gateway events;
- exposing platform resources;
- emitting forwardable events;
- translating Kozz media and mentions into platform-specific calls.

## Helper Library

New boundaries should use `kozz-boundary-maker`. It provides:

- Socket.io connection to the gateway;
- signed introduction payloads;
- event handlers for gateway actions;
- resource response plumbing;
- forwardable event emission;
- inline command parsing helpers.

## Typical Boundary Shape

```ts
import createBoundary from 'kozz-boundary-maker';

const boundary = createBoundary({
  url: process.env.GATEWAY_URL ?? 'ws://localhost:4521',
  socketPath: process.env.SOCKET_PATH,
  chatPlatform: 'MyPlatform',
  name: process.env.BOUNDARY_NAME ?? 'my-boundary',
});

boundary.handleReplyWithText((payload, companion, body) => {
  // Send a text reply through the platform SDK.
});

boundary.onAskResource('chat_status', () => ({
  ready: true,
}));
```

## Current Boundary

The current WhatsApp implementation is [`kozz-baileys`](https://github.com/Kozz-Bot/kozz-baileys), powered by Baileys.

It is an implementation example of the boundary protocol, not the only possible boundary. Other platforms can integrate with Kozz by emitting the same normalized events and handling the same gateway actions.
